# Long Life Talk · 실시간 보드

LG 베스트샵 수원본점 × Long Black 오프라인 커뮤니티 커피챗(2026.06.12)을 위한
실시간 보드입니다. 세 개의 페이지가 하나의 서버로 묶여 실시간 동기화됩니다.

| 페이지 | 경로 | 누가 보나 |
|---|---|---|
| 무대 스크린 | `/stage` | 무대 빔/대형 TV |
| 조별 입력 | `/group` | 각 조 LG 노트북 7대 |
| 어드민 콘솔 | `/admin` | 진행자(사회자 옆) |

조가 노트북에서 문장을 전송 → 어드민에 도착 → 진행자가 대표 답변을 채택 → 무대 스크린에
조별 말풍선으로 노출되는 구조입니다. (자동 노출 모드로 바꾸면 전송 즉시 무대에 뜹니다.)

---

## 1. 빠른 시작

```bash
npm install      # 최초 1회
npm start        # 서버 실행 (기본 포트 3000)
```

실행하면 터미널에 접속 주소가 출력됩니다.

```
로컬:        http://localhost:3000
네트워크:    http://192.168.x.x:3000   ← 같은 와이파이의 다른 기기는 이 주소로 접속
```

- 진행 PC(서버 돌리는 노트북)에서 `http://localhost:3000/stage`, `/admin` 을 엽니다.
- 조별 노트북에서는 위 **네트워크 주소** + `/group` 으로 접속합니다.
- 포트 변경: `PORT=8080 npm start`

---

## 2. 행사 당일 권장 세팅 (★ 가장 중요)

### 방식 A. 현장 로컬 네트워크 (권장 — 인터넷 없어도 됨)
1. 노트북(또는 미니PC) 한 대를 **서버 겸 무대 출력 PC**로 정합니다.
2. 이 PC를 무대 스크린(빔/TV)에 연결하고 `npm start` 실행.
3. 모든 기기(조별 노트북 7대 + 서버 PC)를 **같은 와이파이/공유기**에 연결합니다.
   - 행사장 와이파이가 불안정하면 **휴대용 공유기 1대를 따로 가져가** 전용 네트워크를 만드는 걸 강력 추천합니다. (외부 인터넷 불필요)
4. 조별 노트북은 브라우저에 `http://서버IP:3000/group` 즐겨찾기 또는 시작페이지로 세팅.

> 폰트(Pretendard 등)는 인터넷이 있을 때만 로드되고, 없으면 시스템 폰트로 깔끔하게 폴백되도록 해두었습니다. 기능은 인터넷 없이도 100% 동작합니다.

### 방식 B. 클라우드 배포 (URL 하나로 어디서나)
행사장 인터넷이 안정적이라면 클라우드에 올려 짧은 URL로 접속할 수 있습니다.
Render / Railway / Fly.io 등 Node 앱을 그대로 올리면 됩니다. (Procfile/빌드 설정 불필요, `npm start`만 있으면 됨)

- 예) Render: New Web Service → 이 저장소 연결 → Build `npm install` / Start `npm start`
- 배포 후 `https://내앱.onrender.com/stage`, `/group`, `/admin` 으로 접속.

**리허설은 반드시 6/11 또는 그 전날, 실제 노트북·실제 네트워크로 한 번 돌려보세요.**

## 3. 진행 시나리오 (어드민 콘솔 사용법)

1. **조 개수 설정**: 좌측 `조 개수`를 실제 조 수(기본 7)에 맞춥니다.
2. **조 접속 확인**: 각 조가 `/group`에서 자기 조 번호를 누르면, 어드민의 `조 접속 현황`에 불이 들어옵니다. 7개 조가 다 켜졌는지 확인.
3. **질문 띄우기**: 미리 넣어둔 프리셋을 클릭하거나 직접 입력해 `띄우기`. 무대와 조별 노트북에 동시에 반영됩니다.
4. **답변 받기**: 조가 전송하면 우측 `들어온 답변`에 **조별로** 쌓입니다.
5. **대표 채택**: 동시에 여러 개가 들어와도, 보고 고른 답변의 `무대 채택`을 누르면 그 답변만 무대에 뜹니다.
   - `한 조당 1개만 노출`이 켜져 있으면, 같은 조의 새 답변을 채택할 때 기존 것은 자동으로 내려갑니다.
   - 잘못 올린 건 `내리기`, 오타·부적절 답변은 `삭제`.
6. **다음 질문**: 새 질문을 띄우면 무대가 비워지고 새 라운드가 시작됩니다.

### 모드 선택
- **채택형(moderated)** — 기본값. 진행자가 고른 답변만 무대에 노출. 사회자가 읽어주며 진행하기 좋습니다.
- **자동형(auto)** — 전송하는 즉시 무대에 와르르 뜹니다. 라이브한 분위기를 원할 때.

언제든 어드민에서 모드를 바꿀 수 있습니다.

---

## 4. 자주 묻는 상황

- **조가 새로고침하면?** 처음 고른 조 번호가 기억되어 그대로 입력 화면으로 돌아옵니다. (조 변경 버튼으로 재선택 가능)
- **무대 PC 연결이 잠깐 끊기면?** 자동 재접속됩니다. 재접속되면 현재 무대 상태를 그대로 다시 그립니다.
- **서버를 재시작하면?** 모든 데이터가 초기화됩니다(메모리 저장). 행사 중에는 서버를 끄지 마세요.
- **답변이 너무 길면?** 500자로 제한됩니다. 한 문장 권장.

## 5. 구조

```
long-life-talk/
├── server.js              # Express + Socket.IO 서버 (모든 상태는 메모리)
├── package.json
└── public/
    ├── index.html         # 안내 랜딩
    ├── stage.html         # 무대 스크린
    ├── group.html         # 조별 입력
    ├── admin.html         # 어드민 콘솔
    ├── css/common.css     # 공통 디자인 토큰
    └── js/
        ├── stage.js
        ├── group.js
        └── admin.js
```

상태/색·문구·프리셋 질문 등은 `server.js` 상단과 각 `js` 파일 상단에서 쉽게 바꿀 수 있습니다.
